# 灵讯哨操作文档

软件名：灵讯哨

含义：守在窗口消息旁的提醒哨位，发现新的系统消息后及时通知。
作者：tegic
联系方式：35350826
GitHub：https://github.com/teg1c

可执行文件名：

```text
windows-monitor.exe
```

## 1. 准备环境

### 必需

- Windows 电脑或 Windows 远程桌面环境
- 已安装 .NET SDK 或 .NET Runtime
- 游戏窗口处于可见状态

首次编译会从 NuGet 拉取 AntdUI：

```text
AntdUI 2.4.0
```

### 可选 OCR

二选一即可：

- wxocr API 服务：当前默认地址 `http://192.168.88.3:5000/ocr`
- 本地 OCR 命令：例如 Tesseract、PaddleOCR、RapidOCR，要求命令能把识别文本输出到 stdout

## 4. 选择监控窗口

1. 打开梦幻西游客户端，并让目标窗口保持可见。
2. 在程序里点击“刷新窗口”。
3. 在窗口下拉框里选择梦幻西游窗口。
4. 点击“共享预览”。

右侧会显示所选窗口的实时预览。

## 5. 设置监控区域

程序现在只做关键词监控。选择窗口、整个桌面或某个显示器后，可以在右侧预览图上框选局部区域，也可以不框选。

- 已框选：只 OCR 框选区域，适合盯住图片查看器里的弹窗、游戏聊天区或桌面上的固定区域。
- 未框选：OCR 整个所选来源。如果来源选择“整个桌面”，就是监控当前可见桌面；如果来源选择某个显示器，就是监控该显示器。

框选后，左侧“区域”会显示：

```text
x=..., y=..., w=..., h=...
```

## 6. 设置监控参数

### 预览间隔毫秒

窗口截图检测间隔。建议：

```text
300 - 1000
```

数值越小越实时，但资源占用越高。

### OCR 间隔毫秒

关键词识别间隔。建议：

```text
1500 - 3000
```

数值越小越实时，但 OCR 占用会更高。本地 PaddleOCR 首次加载会慢一些。

### 冷却秒数

两次提醒之间的最短间隔。建议：

```text
5 - 30
```

## 7. 设置 OCR

### 模式一：wxocr API

选择 OCR 模式：

```text
wxocr
```

OCR 地址填写：

```text
http://192.168.88.3:5000/ocr
```

最低置信度建议：

```text
45
```

如果识别不到文字，可以适当降低。若识别出很多错误文字，可以提高。

### 模式二：内置本地 OCR

选择 OCR 模式：

```text
local
```

这个模式使用内置 PaddleOCR ChineseV5 本地模型，不需要启动 wxocr 服务，也不需要安装 Tesseract。首次识别会加载模型，可能会慢一些；后续会复用同一个本地 OCR 引擎。

说明：

- 发布目录会变大，因为包含 PaddleOCR、OpenCV、Paddle Inference 和中文模型
- 适合关键词监控和桌面局部区域监控
- 如果本机性能一般，建议把 OCR 间隔设置为 `2000` 毫秒以上

### 模式三：本地命令

选择 OCR 模式：

```text
command
```

默认 Tesseract 配置：

```text
命令：tesseract
参数："{image}" stdout -l chi_sim+eng --psm 6
```

说明：

- `{image}` 会被程序替换为临时截图路径
- OCR 命令必须把识别结果输出到 stdout
- 如果用 PaddleOCR、RapidOCR，可以写一个 `.bat`、`.ps1` 或 `.exe` 封装，再在这里调用

## 8. 选择监控来源

窗口下拉框第一项是：

```text
整个桌面 [DESKTOP]
显示器 1（主屏） [MONITOR]
显示器 2 [MONITOR]
```

如果要监控桌面上的某块区域，例如图片查看器、系统弹窗、浏览器里的局部内容，可以选择“整个桌面”或某个“显示器”。点击“共享预览”后，再在右侧桌面预览图上框选要监控的区域。

说明：

- `整个桌面` 表示 Windows 当前可见的虚拟屏幕，包含所有显示器
- `显示器 1/2/...` 表示单个物理屏幕
- 如果你使用 Windows 虚拟桌面，只能监控当前正在显示的虚拟桌面；切到其他虚拟桌面后，不可见的桌面内容无法截图

监控页只有关键词监控一种逻辑：

1. 选择窗口、整个桌面或某个显示器。
2. 可选框选区域；不框选就监控整个所选来源。
3. 设置“OCR 间隔毫秒”，建议先用默认 `2000`。
4. 在“命中关键词”里每行写一个关键词。

示例：

```text
网络错误
请重新登录
网络有问题
```

开启后，程序会按 OCR 间隔识别监控画面。只要识别文本包含任意一个关键词，就会发送通知。必须开启 OCR，并且至少配置一个关键词，否则无法开始监控。

## 9. 设置通知

### 托盘提醒

程序默认会使用 Windows 托盘气泡提醒。

### Webhook

如需转发到 Server 酱、Bark、钉钉、飞书等服务：

1. 点击“新增渠道”。
2. 填写渠道名，选择预设渠道，点击“应用预设”。
3. 填入 Webhook 地址。
4. 按需要调整 Method、Headers、Body 模板。
5. 点击“保存渠道”。
6. 在渠道列表中勾选要启用的渠道。可以同时勾选多个，命中关键词后会逐个发送。

支持的预设：

```text
generic
serverchan
bark
dingtalk
feishu
wecom
custom
```

Body 模板可用变量：

```text
{title} {body} {message} {window} {distance} {region} {ocrText} {time} {timestamp}
```

JSON 模板示例：

```json
{
  "title": "梦幻西游系统消息",
  "body": "{body}",
  "createdAt": "{time}",
  "window": "{window}",
  "distance": {distance}
}
```

说明：

- JSON 模板里优先使用 `{body}`、`{title}`，程序会处理 JSON 字符串转义
- URL 或表单参数里使用 `{bodyUrl}`、`{titleUrl}`
- 纯文本模板里使用 `{bodyRaw}`、`{titleRaw}`

## 10. 开始监控

确认以下内容都已设置：

- 已选择窗口
- 已点击“共享预览”
- 已按需要框选区域；不框选时会监控整个所选来源
- 已配置命中关键词，至少一行
- OCR 已开启并配置正确
- 通知渠道已配置

点击：

```text
开始监控
```

程序不会做图像变化判断，只按“OCR 间隔毫秒”识别监控画面。命中任意关键词后提醒，并按“冷却秒数”去重。同一命中内容未变化时最多提醒 3 次；OCR 未命中后会重置计数。没有输入关键词时，程序会阻止开始监控。

## 11. 关闭程序

点击窗口右上角关闭按钮时，会弹出确认框：

- “是”：退出程序
- “否”：最小化，监控继续运行
- “取消”：不关闭，留在当前窗口

## 12. 查看日志

程序会把监控错误和关键事件写入日志。

日志位置：

```text
logs\app.log
```

在界面里打开“日志”页签，可以：

- 查看当前日志
- 刷新日志
- 清空日志
- 设置最大日志行数

建议最大日志行数：

```text
1000 - 5000
```

如果监控中出现 OCR 失败、Webhook 失败、窗口截图失败等问题，优先查看这里。

## 13. 配置文件

程序会自动保存配置到：

```text
dist\config.local.json
```

如果使用 `dotnet run`，配置一般保存到构建输出目录。

常见配置项：

- `pollIntervalMs`: 轮询间隔
- `cooldownSeconds`: 冷却秒数
- `monitorMode`: 当前固定保存为 `keyword`
- `ocrMode`: `wxocr`、`local` 或 `command`
- `ocrUrl`: wxocr API 地址
- `ocrCommand`: 本地 OCR 命令
- `ocrArguments`: 本地 OCR 参数
- `fullWindowKeywordEnabled`: 旧版兼容字段
- `keywordOcrIntervalMs`: OCR 间隔
- `watchKeywords`: 命中关键词，每行一个
- `region`: 框选区域
- `notificationChannels`: 多通知渠道配置
- `maxLogLines`: 最大日志行数

## 14. 常见问题

### 找不到梦幻西游窗口

- 确认游戏窗口已经打开
- 点击“刷新窗口”
- 确认不是最小化状态

### 预览是黑屏或空白

- 确认远程桌面没有断开
- 确认游戏窗口没有最小化
- 尝试把游戏窗口切到前台

### 提醒太频繁

- 缩小框选区域
- 增加冷却秒数

### 没有提醒

- 确认已点击“开始监控”
- 确认 OCR 有识别结果
- 确认已配置关键词，未输入关键词时程序不会开始监控
- 降低 wxocr 最低置信度
- 检查 Webhook 地址是否可用
- 查看“日志”页签里的错误信息

### OCR 识别不准

- 调整框选区域，只包含文字
- wxocr 模式下调整最低置信度
- 本地命令模式下尝试 PaddleOCR 或 RapidOCR
- Tesseract 模式下确认安装了中文语言包 `chi_sim`

## 15. 使用边界

本程序只做外部窗口截图和 OCR 识别：

- 不注入游戏进程
- 不读取游戏内存
- 不安装键鼠钩子
- 不隐藏或伪装进程
- 不绕过检测

请只在游戏规则允许的范围内使用。
